# 小米商城前端实训一周开发计划书

## 一、项目基本信息

项目名称：小米商城前端仿站项目  
项目类型：静态前端页面展示项目  
开发周期：1 周  
小组人数：5 人  
开发工具：VS Code、浏览器、GitHub 或 Gitee  

技术范围：

- HTML5：负责页面结构搭建。
- CSS3：负责页面布局、美化和响应式适配。
- JavaScript：负责基础页面交互。
- GitHub 或 Gitee：负责小组代码管理和协作提交。

项目边界：

- 本项目只完成前端页面展示和基础交互。
- 不使用后端框架。
- 不连接数据库。
- 登录页只做页面展示和前端表单校验，不做真实账号验证。
- 购物车只做前端数量、小计和总价演示，不涉及真实订单和支付。

## 二、项目目标

本项目以小米商城为参考对象，完成一个简化版前端商城网站。项目最终至少包含 5 个内容完整的页面，并通过导航栏、按钮或超链接实现页面之间的跳转。

最终成果应包括：

```text
xiaomi-shop/
├─ index.html       首页
├─ products.html    商品列表页
├─ detail-*.html    商品详情页
├─ cart.html        购物车页
├─ login.html       登录页
├─ css/
│  ├─ common.css    公共样式
│  ├─ index.css     首页样式
│  ├─ products.css  商品列表页样式
│  ├─ detail.css    商品详情页样式
│  ├─ cart.css      购物车页样式
│  └─ login.css     登录页样式
├─ js/
│  ├─ main.js       公共交互脚本
│  ├─ products.js   商品列表页脚本
│  ├─ detail.js     商品详情页脚本
│  ├─ cart.js       购物车页脚本
│  └─ login.js      登录页脚本
└─ images/          图片素材
```

最终项目应能够在浏览器中打开首页，并通过导航栏或页面按钮进入商品列表页、商品详情页、购物车页和登录页。

## 三、页面功能规划

### 1. 首页 `index.html`

主要内容：

- 顶部导航栏
- 搜索区域
- 商品分类入口
- 宣传 Banner 区域
- 热门商品推荐
- 服务保障或选购说明
- 页脚信息

说明：

计划书中的“轮播图”不作为必须功能。本项目可以使用固定宣传 Banner 区域展示主推商品，这样更贴合当前项目成果，也便于小组成员讲解。

主要技术点：

- HTML 语义化结构
- Flex 或 Grid 页面布局
- 商品卡片样式
- 页面跳转链接

### 2. 商品列表页 `products.html`

主要内容：

- 商品列表标题
- 商品分类筛选区域
- 至少 8 个商品卡片
- 商品图片、名称、简介、价格
- 查看详情按钮

主要技术点：

- Grid 商品网格布局
- 商品卡片 hover 效果
- 分类按钮高亮或筛选显示
- 跳转到商品详情页

### 3. 商品详情页 `detail-*.html`

主要内容：

- 商品大图
- 商品名称
- 商品简介
- 商品价格
- 版本选择
- 颜色选择
- 加入购物车按钮
- 商品参数说明

主要技术点：

- 左右分栏布局
- 版本和颜色按钮选中效果
- 加入购物车提示效果

### 4. 购物车页 `cart.html`

主要内容：

- 购物车商品列表
- 商品图片和商品信息
- 商品单价
- 数量加减按钮
- 单个商品小计
- 购物车总价
- 继续购物和结算按钮

主要技术点：

- 购物车列表布局
- JavaScript 实现数量加减
- JavaScript 实现小计和总价计算

可选扩展功能：

- 删除购物车商品。

说明：

删除商品功能不作为必须验收内容。如果时间充足，可以作为扩展功能完成；如果时间紧张，只需要保证数量加减、小计和总价计算正常。

### 5. 登录页 `login.html`

主要内容：

- 登录表单
- 用户名或手机号输入框
- 密码输入框
- 登录按钮
- 注册入口提示
- 错误提示文字

主要技术点：

- 表单布局
- 输入框和按钮样式
- JavaScript 前端校验

说明：

登录页只需要做前端展示和基础表单校验。校验内容包括用户名是否为空、密码是否为空、密码长度是否符合要求。输入符合要求后显示“校验通过”或类似提示，不需要连接后端数据库。

- 实现颜色按钮选中效果。

商品详情页验收标准：

## 六、一周开发进度安排

### 第 1 天：需求分析和项目准备

全组任务：

1. 明确项目主题为小米商城前端仿站。
2. 明确项目只做静态前端页面。
3. 明确 5 个页面和每个人负责的模块。
4. 建立项目目录。
5. 收集商品图片和商品文字信息。
6. 学习 GitHub 或 Gitee 基本协作方式。

当天成果：

- 项目目录建立完成。
- 小组分工明确。
- 页面清单和文件结构确定。
- 每个成员知道自己负责的页面和交付内容。

### 第 2 天：公共结构和首页开发

全组任务：

1. 统一导航栏。
2. 统一页脚。
3. 统一按钮、字体、颜色和页面宽度。
4. 每个成员开始完成自己页面的 HTML 结构。

当天成果：

- 首页初步完成。
- 其他页面 HTML 结构初步完成。
- 公共导航栏和页脚基本统一。

### 第 3 天：CSS 布局和页面美化

全组任务：

1. 每个成员完成自己页面的 CSS。
2. 保证页面主色调、按钮、字体和卡片样式统一。
3. 检查页面是否有明显错位。

当天成果：

- 所有页面基本美化完成。
- 页面风格基本统一。
- 静态页面可以正常展示。

### 第 4 天：JavaScript 交互开发

全组任务：

1. 完成必要的基础交互。
2. 避免实现过于复杂的功能。
3. 每个成员只完成自己页面需要的 JavaScript。

成员任务：

- A：检查首页链接和整体页面跳转。
- B：完成商品分类按钮高亮或商品筛选效果。
- C：完成版本选择、颜色选择和加入购物车提示。
- D：完成购物车数量加减、小计和总价计算。
- E：完成登录表单前端校验。

当天成果：

- 页面基础交互完成。
- 购物车和登录页功能可演示。
- 商品详情页选项可点击。

### 第 5 天：代码合并和联调

全组任务：

1. 每个成员提交自己的代码。
2. 组长合并代码。
3. 全组检查页面跳转。
4. 修复明显的样式冲突和 JavaScript 报错。

当天成果：

- 所有页面合并到同一个项目。
- 页面之间可以正常跳转。
- 主要功能可以使用。

### 第 6 天：测试、修改和文档整理

全组任务：

1. 根据验收清单检查项目。
2. 修复页面显示问题。
3. 检查浏览器控制台是否有 JavaScript 报错。
4. 每个成员整理自己负责模块的实训日志。
5. E 负责汇总测试记录，组长协助整理报告。

当天成果：

- 项目基本达到验收标准。
- 每个成员完成自己的实训日志。
- 测试记录初步完成。

### 第 7 天：最终验收和演示准备

全组任务：

1. 完成最终版本。
2. 完成实训报告。
3. 准备验收演示。
4. 每个成员准备讲解自己负责的页面。

当天成果：

- 项目最终版本完成。
- 实训报告完成。
- 每个成员都能讲清楚自己负责的内容。

推荐提交信息：

```text
完成首页导航栏和 Banner 区域
完成商品列表页 HTML 结构
完成商品详情页版本选择功能
完成购物车数量加减和总价计算
完成登录页表单校验
修复商品卡片图片大小不一致问题
```

## 八、最终验收检查清单

项目提交前，全组一起检查：

1. 是否包含至少 5 个页面。
2. 首页内容是否完整。
3. 页面之间是否可以正常跳转。
4. 商品列表页是否有不少于 8 个商品。
5. 商品详情页是否有完整商品信息。
6. 购物车是否可以数量加减。
7. 购物车小计和总价是否能自动计算。
8. 登录页是否有前端表单校验。
9. 登录页是否没有连接后端数据库。
10. 页面风格是否统一。
11. 图片路径是否正确。
12. CSS 是否正常加载。
13. JavaScript 是否正常运行。
14. 浏览器控制台是否没有明显报错。
15. 实训日志是否完整。
16. 实训报告是否包含项目目的、开发内容、开发过程、问题分析和总结。

1. 不要所有成员同时直接修改主分支。
2. 不要随便修改其他成员负责的页面。

7. 每个成员都要保留自己的实训日志，不能只由组长一个人完成。
