# 小米商城前端仿站实训计划书

## 一、项目名称

小米商城前端页面仿站设计与实现

## 二、项目目标

本项目以小米商城为参考对象，使用 HTML、CSS 和 JavaScript 完成一个简化版商城网站。网站需要包含不少于 5 个内容较完整的页面，并通过超链接实现页面之间的跳转，形成一个基本完整的前端网站。

本项目重点训练页面结构设计、CSS 布局、美化、JavaScript 基础交互、项目协作和实训文档撰写能力。

## 三、技术方案

- 页面结构：HTML5
- 页面样式：CSS3
- 页面布局：Flex 布局、Grid 布局
- 页面交互：原生 JavaScript
- 开发工具：VS Code
- 浏览器测试：Edge 或 Chrome

本项目不使用 Vue、React 等前端框架，避免超出实训基础要求，保证代码结构清晰、易于讲解和验收。

## 四、页面规划

### 1. 首页 index.html

主要内容：
- 顶部导航栏
- 搜索框
- 商品分类入口
- 轮播图或宣传图区域
- 热门商品推荐
- 手机、家电、智能设备等商品展示区
- 页脚信息

主要技术点：
- HTML5 语义标签
- CSS 布局
- 商品卡片样式
- 简单轮播或图片切换效果

### 2. 商品列表页 products.html

主要内容：
- 分类导航
- 商品筛选区域
- 商品卡片列表
- 商品名称、图片、价格、简介
- 跳转商品详情按钮

主要技术点：
- Grid 商品网格布局
- hover 悬停效果
- 页面跳转

### 3. 商品详情页 detail.html

主要内容：
- 商品大图
- 商品名称
- 商品价格
- 商品简介
- 版本选择
- 颜色选择
- 加入购物车按钮
- 商品参数说明

主要技术点：
- 左右分栏布局
- 按钮选中效果
- JavaScript 实现版本和颜色选择状态

### 4. 购物车页 cart.html

主要内容：
- 购物车商品列表
- 商品单价
- 数量加减
- 小计金额
- 总价计算
- 删除商品按钮

主要技术点：
- 表格或列表布局
- JavaScript 数量加减
- JavaScript 总价计算

### 5. 登录页 login.html

主要内容：
- 登录表单
- 手机号或用户名输入框
- 密码输入框
- 登录按钮
- 注册入口提示

主要技术点：
- 表单布局
- 输入框样式
- JavaScript 空值校验

## 六、小组分工建议

### 组长

负责整体协调、页面风格统一、进度检查、最终验收演示和报告整合。

### 组员一

负责首页 `index.html`，包括导航、轮播、商品推荐和页脚。

### 组员二

负责商品列表页 `products.html`，包括分类区域、商品卡片和详情页跳转。

### 组员三

负责商品详情页 `detail.html`，包括商品信息展示、版本颜色选择和加入购物车按钮。

### 组员四

负责购物车页 `cart.html` 和登录页 `login.html`，包括购物车数量计算、总价计算和登录表单校验。

如果实际小组人数不足，可以将“商品详情页”和“购物车、登录页”合并给同一人完成。

## 七、开发进度安排

### 第一天：需求分析与项目准备

- 确定选题为小米商城
- 分析参考网站页面结构
- 确定 5 个页面
- 建立项目目录
- 准备图片和文字素材

### 第二天：首页开发

- 完成公共导航栏
- 完成首页主体布局
- 完成商品推荐区域
- 初步统一页面颜色和字体

### 第三天：商品列表页与详情页开发

- 完成商品列表页
- 完成商品详情页
- 实现页面之间的跳转
- 完成基础商品信息展示

### 第四天：购物车页与登录页开发

- 完成购物车页面布局
- 实现数量加减和总价计算
- 完成登录页面
- 实现表单空值校验

### 第五天：整合测试与实训文档

- 检查所有页面跳转
- 检查样式统一性
- 修复显示问题和 JavaScript 报错
- 整理实训日志
- 编写实训报告
- 准备验收演示

## 八、协作规范

### 文件命名规范

- HTML 文件使用英文小写命名
- CSS 文件统一写在 `css/style.css`
- JavaScript 文件统一写在 `js/main.js`
- 图片放入 `images/` 文件夹

### 代码规范

- HTML 标签缩进清晰
- CSS 类名尽量表达模块含义
- 公共样式集中管理
- 不同页面保持统一导航和页脚
- JavaScript 只实现必要交互，避免复杂逻辑

### 页面风格规范

- 主色调参考小米商城的橙色、白色、浅灰色
- 商品卡片保持统一尺寸
- 按钮样式保持一致
- 页面间导航位置保持一致

## 九、验收检查清单

- 是否包含至少 5 个页面
- 页面之间是否能正常跳转
- 首页内容是否丰富
- 商品列表是否有多个商品
- 商品详情页是否有完整商品信息
- 购物车是否能进行数量加减和总价计算
- 登录页是否有表单校验
- CSS 样式是否统一
- 页面是否存在明显错位
- 控制台是否存在 JavaScript 报错
- 实训日志是否完整
- 实训报告是否包含目的、内容、过程、问题分析、总结

## 十、与 Codex 对接方式

后续开发时，每次对接建议按以下格式说明问题：

```text
当前文件：
例如 index.html / css/style.css / js/main.js

我想实现：
例如 首页商品卡片一行显示 4 个

现在的问题：
例如 商品卡片换行错乱，图片大小不一致

我已经写的代码：
粘贴相关 HTML/CSS/JS 片段
```

Codex 可以协助完成：
- 检查 HTML 结构是否合理
- 修改 CSS 布局问题
- 排查 JavaScript 报错
- 优化页面跳转
- 整理实训日志和报告
- 按验收清单做最终检查

为了符合实训要求，Codex 主要作为辅助检查和讲解工具使用，不直接代替小组完成全部页面。

## 十一、预期成果

最终提交内容包括：
- 一个小米商城前端仿站项目文件夹
- 至少 5 个 HTML 页面
- 一个 CSS 样式文件
- 一个 JavaScript 交互文件
- 图片素材文件夹
- 实训日志
- 实训报告

项目完成后，应能够在浏览器中打开首页，并通过导航或按钮进入商品列表页、详情页、购物车页和登录页。
